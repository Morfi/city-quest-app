# Решение проблем

## Ошибка "EMFILE: too many open files"

Эта ошибка возникает на macOS когда система достигает лимита открытых файловых дескрипторов.

### Решение 1: Установить Watchman (рекомендуется)

Watchman - это инструмент от Facebook для мониторинга файлов, который более эффективно управляет файловыми дескрипторами:

```bash
# Установка через Homebrew
brew install watchman

# Или через MacPorts
sudo port install watchman
```

После установки перезапустите Metro bundler.

### Решение 2: Увеличить лимит файловых дескрипторов

Временно увеличить лимит для текущей сессии:

```bash
ulimit -n 4096
```

Для постоянного решения добавьте в `~/.zshrc` или `~/.bash_profile`:

```bash
ulimit -n 4096
```

Затем перезагрузите терминал или выполните:
```bash
source ~/.zshrc
```

### Решение 3: Очистить кэш и перезапустить

```bash
# Очистить кэш Metro
npx expo start --clear

# Или полностью очистить
rm -rf node_modules
npm install
npx expo start --clear
```

### Решение 4: Использовать watchman (если установлен)

```bash
watchman watch-del-all
npx expo start --clear
```

## Несовместимость версий

Если видите предупреждения о несовместимости версий:

```bash
npx expo install --fix
```

Это автоматически обновит все зависимости до совместимых версий.
